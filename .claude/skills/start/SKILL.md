Ты помогаешь студенту выполнить лабораторную работу. Сейчас — этап старта.

## Что делать

1. Прочитай файлы проекта: CLAUDE.md (требования к оформлению), task.md (задание лабораторной работы).
2. Изучи все остальные файлы в директории проекта — возможно, есть дополнительные материалы. Файл вида ЛР_N__БИВТ-24-2_... — это личный файл пользователя, не открывай его.
3. Если что-то в задании неясно или тебе не хватает знаний — выполни поиск в интернете для уточнения. Не выдумывай.
4. Сформируй файл instructions.md — пошаговую инструкцию для пользователя.

## Требования к instructions.md

- Максимальная подробность: какую программу открыть, что нажать, какую команду ввести, на что обратить внимание.
- Markdown-форматирование допустимо.
- В местах, где пользователю нужно сделать скриншот, укажи это явно: какое окно, что на нём должно быть. Формат подписи: Рисунок N. Описание (нумерация сквозная).
- В местах, где пользователь должен вставить результат выполнения команды/действия, ставь плейсхолдер: [ВСТАВИТЬ РЕЗУЛЬТАТ СЮДА].
- Учитывай доступную периферию пользователя (описана в CLAUDE.md).
- Если задание связано с сетью: используй реальные данные из секции «Сетевые данные устройств:» в CLAUDE.md — подставляй конкретные IP-адреса, MAC-адреса, имена хостов вместо абстрактных примеров. Если данные не настроены — предупреди пользователя, что сначала нужно запустить /setup.
- Если задание подразумевает мало скриншотов — предложи в нужных местах разместить картинки из интернета. Укажи поисковый запрос и описание нужной картинки.

## Дополнительный контекст от пользователя

$ARGUMENTS

## После создания

Сообщи пользователю, что instructions.md готов. Напомни, что далее он следует инструкции, заполняет результаты, а при проблемах может вызвать /assist. Напомни еще, что когда он закончит, он может использовать следующую по плану команду - /report.
