Пользователь хочет настроить проект под себя. Нужно узнать, какие устройства у него есть и есть ли пожелания к отчётам/инструкциям, а затем записать это в CLAUDE.md.

## Что делать

### Шаг 1. Опрос о периферии

Задай пользователю вопросы с помощью инструмента AskUserQuestion. Задай все вопросы одним вызовом:

1. **Основной компьютер.** Какой у тебя основной компьютер и какая на нём ОС? (например: ноутбук на Windows 11, MacBook на macOS Sequoia, ПК на Ubuntu 24.04).
2. **Дополнительные компьютеры.** Есть ли дополнительные компьютеры/ноутбуки? Если да — укажи модель и ОС. Если нет — выбери «Нет».
3. **Телефоны.** Есть ли телефоны, которые могут понадобиться для лабораторных? Если да — укажи модели. Если нет — выбери «Нет».
4. **Сеть.** Все устройства подключены к одной домашней Wi-Fi сети?

### Шаг 2. Опрос о пожеланиях

После получения ответов о периферии задай ещё один вопрос:

- Есть ли у тебя пожелания к оформлению отчётов или инструкций? Например: особый стиль изложения, дополнительные разделы, предпочтения по объёму, конкретные требования преподавателя и т.д. Если нет — выбери «Нет, стандартные настройки».

### Шаг 3. Обновление CLAUDE.md

1. Прочитай текущий CLAUDE.md.
2. Найди секцию «Периферия, доступная пользователю:» и замени её содержимое на данные, полученные от пользователя. Формат:
   ```
   Периферия, доступная пользователю:
   - Основной компьютер: <ответ>.
   - Дополнительные компьютеры: <ответ> (или «Нет»).
   - Телефоны: <ответ> (или «Нет»).
   - Сеть: <ответ>.
   ```
3. Если у пользователя есть пожелания — добавь после секции «Периферия» новую секцию:
   ```
   Пожелания пользователя к отчётам/инструкциям:
   - <пожелание 1>
   - <пожелание 2>
   ...
   ```
   Если пожеланий нет — не добавляй эту секцию (или удали, если она уже существовала).

### Шаг 4. Подтверждение

Покажи пользователю, что было записано в CLAUDE.md (секция периферии и, если есть, секция пожеланий). Сообщи, что настройка завершена и теперь все команды (/start, /report и т.д.) будут учитывать его окружение.
