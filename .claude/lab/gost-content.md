# Требования ГОСТ: стиль написания текста

Этот файл содержит правила написания текстов для отчётов и инструкций по ГОСТ.
Читай при работе с report.md и instructions.md.

## Язык и общие правила

- Все документы пишутся на русском языке.
- Буква **ё** обязательна везде, где она нужна (ё ≠ е).
- Кавычки: «ёлочки», не "лапки".
- Если не хватает знаний — ищи в авторитетных интернет-источниках. Не выдумывай.

## Подписи рисунков и таблиц

- Рисунки подписываются **под рисунком**, выравнивание по левому краю:
  `Рисунок 1. Название рисунка`
- Таблицы подписываются **над таблицей**, выравнивание по правому краю:
  `Таблица 1. Название таблицы`
- Ссылки на рисунки и таблицы в тексте:
  - `Результат выполнения команды представлен на Рисунке 1.`
  - `Выполним команду в терминале (Рисунок 1).`

## Аббревиатуры

- Все аббревиатуры расшифровываются при **первом** упоминании.
  Пример: `Протокол передачи гипертекста (HTTP, HyperText Transfer Protocol) используется для...`

## Пробелы и форматирование

- Обязательные пробелы:
  - при указании адреса: `ул. Воронцовская, д. 6а, стр. 1`
  - между знаком номера и цифрой: `школа № 75`
  - в нумерованном списке после номера:
    ```
    1) вариант «А»;
    2) вариант «В»;
    3) вариант «С».
    ```

## Даты

- Формат: ДД.ММ.ГГГГ или словами.
  Примеры: `25.09.2019`, `25 сентября 2019 г.`

## Тире и дефис

- **Тире** (длинное –) используется в предложениях:
  `Россия – самая большая страна по площади в мире.`
  `Порт 80 закреплён за HTTP, 443 – за HTTPS.`
- **Дефис** (-) используется для указания промежутка или в сложных словах:
  `светло-серый`, `7-10 мс`

## Знак номера

- Только **№**, не латинская N.

## Числа

- Разделитель дробной части: **запятая** (1,9).

## Плейсхолдеры

Единый формат для всех работ:
- Место для вставки картинки/скриншота: `[ВСТАВИТЬ РИСУНОК N ЗДЕСЬ]` (где N – номер рисунка).
- Место для вставки результата выполнения команды/действия (только в инструкции): `[ВСТАВИТЬ РЕЗУЛЬТАТ СЮДА]`.

## Стилистика

- Избегать формулировок типа «Вывод: ...».
  Вместо этого использовать: «Таким образом, ...», «Получили, что ...», «В ходе работы установлено, что ...» и подобные варианты.

## Отчёт: содержание

- Цель работы сформулирована в начале.
- Все пункты задания выполнены и описаны подробно.
- Последовательность действий описана в деталях.
- Результаты объяснены, на возникающие вопросы даны ответы.
- Умозаключения по ходу отчёта (не только в конце).
- Содержательный итоговый вывод (не отписка).
- Выводы соответствуют сформулированной цели.
- Чем подробнее отчёт – тем лучше, но без воды.

## Инструкция: содержание

- В инструкции (instructions.md) markdown-форматирование допустимо.
- Указывать пользователю, когда нужно сделать скриншот (какого окна, что на нём должно быть изображено).
- Скриншоты нужны для каждой введённой команды и каждого действия (если на скриншоте будет новая информация).
- Если тема не подразумевает много скриншотов — предложить пользователю разместить картинки из интернета. Указать поисковый запрос и описание нужной картинки.

## Показатели оценивания отчётов

1. Оформление по ГОСТ.
2. Соблюдение последовательности и содержания материала заданию.
3. Достаточный объём: все пункты задания выполнены, таблицы/рисунки/скриншоты на месте, анализ материала присутствует.
4. Цель работы сформулирована, выводы ей соответствуют.
5. Дополнительные исследования (если задано): не просто изложение фактов, а анализ полученных результатов.
6. Умозаключения по ходу отчёта и содержательный итоговый вывод (не отписка).
